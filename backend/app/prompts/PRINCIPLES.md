# Научные принципы дизайна промптов ExamAI

## Введение

Этот документ описывает научную основу для создания AI промптов в ExamAI. Все промпты должны базироваться на исследованиях в области когнитивной психологии и педагогического дизайна, а не на интуиции.

**Основная цель:** Создавать учебные материалы, которые максимизируют долгосрочное запоминание и глубину понимания.

---

## Теоретическая база

### 1. Cognitive Load Theory (CLT) — John Sweller

**Ключевая идея:** Рабочая память человека ограничена — она может удерживать одновременно только 3-5 единиц информации.

**Типы когнитивной нагрузки:**
- **Intrinsic Load** (внутренняя) — сложность самого материала (не контролируем)
- **Extraneous Load** (посторонняя) — шум, плохое форматирование, избыточная информация (❌ УБИРАЕМ)
- **Germane Load** (продуктивная) — усилия на создание ментальных схем (✅ УСИЛИВАЕМ)

**Как применяем:**
- **Chunking** — разбиваем контент на блоки по 150-300 слов
- **Signaling** — выделяем ключевые термины жирным шрифтом (НЕ целые предложения)
- **Segmentation** — используем заголовки H3/H4 для навигации

**Источники:**
- Sweller, J. (1988). "Cognitive load during problem solving"
- Mayer, R. E. (2009). "Multimedia Learning"

---

### 2. Generative Learning Theory — Richard Mayer

**Ключевая идея:** Пассивное чтение неэффективно. Студент должен **активно генерировать** связи между новой информацией и существующими знаниями.

**Генеративные стратегии:**
1. **Summarizing** — переформулирование своими словами
2. **Mapping** — создание диаграмм связей
3. **Drawing** — визуализация концепций
4. **Imagining** — создание мысленных образов
5. **Self-Testing** — извлечение информации из памяти
6. **Self-Explaining** — объяснение материала себе

**Как применяем:**
- Вопросы для самопроверки **без ответа сразу под ними** (active recall)
- Резюме, которое **переформулирует**, а не копирует текст
- Просьба привести **собственные примеры** (в Tutor промпте)

**Источники:**
- Fiorella, L., & Mayer, R. E. (2016). "Eight Ways to Promote Generative Learning"

---

### 3. Testing Effect (Retrieval Practice)

**Ключевая идея:** Извлечение информации из памяти (тестирование) более эффективно для долгосрочного запоминания, чем повторное чтение.

**Данные:**
- Тестирование повышает запоминание на 30-50% по сравнению с повторным чтением
- Эффект сохраняется даже через месяцы

**Как применяем:**
- Встроенные вопросы для самопроверки после каждых 300-500 слов (Executor)
- Flashcards с откладыванием ответа (Quiz)
- MCQ с объяснениями **почему неправильные ответы неправильны** (обучающее тестирование)

**Источники:**
- Roediger, H. L., & Karpicke, J. D. (2006). "Test-Enhanced Learning"

---

### 4. Advance Organizers — David Ausubel

**Ключевая идея:** Мозг лучше усваивает информацию, если у него уже есть "ячейки", в которые ее можно положить.

**Как работает:**
- Перед деталями даем "карту местности"
- Активируем предварительные знания
- Создаем контекст для новой информации

**Как применяем:**
- Блок **"🎯 Ключевые понятия"** в начале каждого топика (Executor)
- **Learning Outcomes** в начале плана курса (Planner)
- Outline перед детальным контентом (Analyzer)

**Источники:**
- Ausubel, D. P. (1960). "The use of advance organizers in the learning and retention of meaningful verbal material"

---

### 5. Spacing Effect & Interleaving

**Ключевая идея:** Распределенное обучение (с интервалами) эффективнее массированного («зубрежки»).

**Как применяем:**
- Flashcards разных уровней сложности (40% easy, 40% medium, 20% hard) — для future spacing
- MCQ разных типов (recall, application, analysis) — interleaving
- **Примечание:** Сам spacing реализуется на уровне приложения (review scheduler), но промпты должны создавать контент, пригодный для spacing

**Источники:**
- Cepeda, N. J., et al. (2006). "Distributed practice in verbal recall tasks: A review and quantitative synthesis"

## Практические принципы для промптов

### ✅ DO: Применяйте эти техники

#### Форматирование (снижение Extraneous Load)
- ✅ Короткие параграфы: 2-4 предложения
- ✅ Заголовки H3 для каждой подтемы
- ✅ Буллеты для списков (не стены текста)
- ✅ Таблицы для сравнений
- ✅ Жирный шрифт **только** для key terms (первое упоминание)
- ✅ Horizontal rules `---` между крупными блоками

#### Структура (Advance Organizers)
- ✅ Начинайте с краткого обзора "Что вы узнаете"
- ✅ Блок "Ключевые понятия" перед деталями
- ✅ Логическая прогрессия: определение → объяснение → пример

#### Активное вовлечение (Generative Learning)
- ✅ Вопросы для самопроверки каждые 300-500 слов
- ✅ Вопросы требуют **понимания**, не простого recall
- ✅ Резюме должно **переформулировать**, не копировать
- ✅ Просите примеры из реальной жизни (где уместно)

#### Тестирование (Testing Effect)
- ✅ Flashcards: открытые вопросы ("Объясни...", "Как..."), не "Что такое X?"
- ✅ MCQ: объясняйте **почему** правильный ответ правильный
- ✅ MCQ: объясняйте **почему** ключевые дистракторы неправильные

---

### ❌ DON'T: Избегайте этих ошибок

#### Когнитивная перегрузка
- ❌ Стены текста без заголовков (>5 предложений подряд)
- ❌ Выделение жирным **целых предложений** (визуальный шум)
- ❌ Избыточное использование эмодзи (>2-3 на экран)
- ❌ Сложные предложения с вложенными конструкциями

#### Пассивное обучение
- ❌ Просто изложение фактов без вопросов
- ❌ Резюме, которое копирует текст выше
- ❌ Примеры без объяснения "Почему это важно"

#### Технические ошибки
- ❌ Плейсхолдеры `[Insert Definition Here]` — ВСЕГДА генерируйте реальный контент
- ❌ Противоречивые инструкции (сначала "будь кратким", потом "детально объясни")
- ❌ Неопределенные требования ("сделай хорошо")

## Chain-of-Thought Prompting

**Ключевая идея:** Заставить модель "думать вслух" перед генерацией финального ответа.

**Когда применять:**
- Сложные структурированные задачи (Executor, Planner)
- Генерация вопросов (Quiz)
- Анализ контента (Analyzer)

**Пример:**
```
Before generating the content, first silently outline:
1. The 3-5 key concepts to cover
2. One self-check question for each concept
3. How these concepts connect to each other

Then generate the formatted study notes.
```

**Зачем это работает:** LLM лучше справляются с задачами, если сначала планируют, а потом выполняют (multi-step reasoning).

## Mode-Switching для Tutor

**Проблема:** Сократический метод ("отвечать вопросом на вопрос") может раздражать студента, который просто хочет быстрый ответ.

**Решение:** Определять интент пользователя:

| Интент | Индикаторы | Режим |
|--------|-----------|-------|
| **Глубокое понимание** | "Help me understand...", "Why does...", "I'm confused about..." | Socratic questioning |
| **Быстрый ответ** | "Quick definition of...", "What is X?", "I have exam in 1 hour" | Direct answer + 1 check question |
| **Практика** | "Give me practice problems", "Test me on..." | Generate quiz/flashcards |

**Реализация:** В промпте `tutor/chat_system.txt`

## Orphan Knowledge Prevention

**Проблема:** Изолированные факты ("Париж — столица Франции") плохо запоминаются без контекста.

**Решение для Flashcards:**
- **Front:** "Почему Париж стал политическим центром Франции?" (контекст)
- **Back:** "Париж — столица с X века из-за [исторические причины]..."

**НЕ:**
- **Front:** "Столица Франции?"
- **Back:** "Париж"

**Принцип:** Каждый факт должен быть **связан** с более широкой концепцией.

## Чек-лист проверки промпта

Перед добавлением нового промпта, убедитесь:

### Структура
- [ ] Есть четкое описание роли AI ("You are a...")
- [ ] Указан контекст и цель
- [ ] Есть конкретные инструкции (не "Сделай хорошо", а "Используй H3 для подзаголовков")
- [ ] Есть примеры форматов (где уместно)

### Педагогические принципы
- [ ] **Chunking:** Контент разбит на блоки <300 слов?
- [ ] **Signaling:** Жирный шрифт только для ключевых терминов?
- [ ] **Advance Organizer:** Есть предварительная структура?
- [ ] **Active Recall:** Есть вопросы для самопроверки?
- [ ] **Generative Learning:** Требуется переформулирование?

### Когнитивная нагрузка
- [ ] Инструкции понятны с первого прочтения
- [ ] Нет противоречивых требований
- [ ] Нет избыточной информации

## Версионирование

При изменении промптов:
1. Документируйте изменения в git commit message
2. Указывайте, какой принцип применяется/улучшается
3. Тестируйте на sample data перед деплоем

**Пример commit message:**
```
feat(prompts): add active recall questions to executor

- Insert self-check questions every 300-500 words (Testing Effect)
- Questions require understanding, not simple recall
- No answer provided below question (forces retrieval)

Refs: Roediger & Karpicke (2006)
```
