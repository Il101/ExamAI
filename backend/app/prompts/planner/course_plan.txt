## Scientific Framework

**Why structure matters:**

1. **Cognitive Load Theory (Chunking)** — Breaking content into 20-30 minute blocks prevents working memory overload (Sweller, 1988).

2. **Advance Organizers** — Knowing the learning outcomes BEFORE diving into content activates prior knowledge and improves encoding (Ausubel, 1960).

3. **Bloom's Taxonomy** — Progression from foundational concepts (Remember/Understand) to advanced applications (Apply/Analyze) creates optimal learning paths.

---

## Your Task

### 1. Identify Real Subject

Read the materials and determine:
- What is the ACTUAL subject/domain? (Not the filename, but the content)
- What level of expertise does this assume? (Beginner, Intermediate, Advanced)

### 2. Extract Learning Outcomes

Before creating blocks/topics, identify **3-6 high-level Learning Outcomes** that answer: "After studying this material, the student will be able to..."

**Format:**
- Action verbs (Bloom's Taxonomy): Explain, Apply, Analyze, Evaluate, Create
- Specific and measurable
- Cover the breadth of the content

**Examples:**
- ✅ "Explain how TCP ensures reliable data transmission"
- ✅ "Apply object-oriented principles to design a class hierarchy"
- ❌ "Understand networking" (too vague)
- ❌ "Learn about programming" (not measurable)

### 3. Divide into Logical Blocks

**Blocks** = Coherent chapters/modules (e.g., "Introduction to Python", "Control Flow Structures")

**Criteria:**
- Each block should be completable in one study session (20-30 min)
- Aim for **3-6 blocks total** (cognitive chunking)
- Keep blocks balanced in size (similar number of topics per block)
- Logical progression: fundamentals → intermediate → advanced

### 4. Define Topics within Each Block

**Topics** = Specific concepts within a block (e.g., "Variables and Data Types", "If-Else Statements")

**Criteria:**
- Add as many topics as necessary to cover the material thoroughly
- Each topic = one focused concept or skill
- Base ONLY on content that exists in the materials (no invention)
- Estimate content size (3-8 paragraphs per topic)

## Planning Strategy

### Block Structure Pattern

**Recommended progression:**

1. **Block 1: Foundations**
   - Basic definitions
   - Core terminology
   - Fundamental concepts
   - (~2-3 topics)

2. **Block 2-3: Core Content**
   - Main concepts/skills
   - Mechanisms/processes
   - Examples and applications
   - (~3-4 topics each)

3. **Block 4 (optional): Advanced/Synthesis**
   - Complex scenarios
   - Integration of concepts
   - Real-world applications
   - (~2-3 topics)

**Total: 3-6 blocks, topics as needed**

### Logical Flow Within Blocks

Each block should progress: **Concrete → Abstract**, **Simple → Complex**

Example (Python Block):
1. Variables and Assignment (concrete, simple)
2. Data Types (concrete, simple)
3. Type Conversion (abstract, medium)
4. Scope and Lifetime (abstract, complex)

---

## Edge Cases

### Content is too short for multiple blocks
- Create 1-2 blocks with fewer topics
- Don't artificially inflate; be honest about scope

### Content covers multiple unrelated subjects
- Focus on the PRIMARY subject (>60% of content)
- Mention secondary topics in learning outcomes if relevant

### Content is disorganized/lecture transcript
- Impose logical structure (even if source doesn't have one)
- Group related concepts together
- Create foundational → advanced progression
