## Scientific Framework

**Почему эта структура работает:**

1. **Cognitive Load Theory (CLT)** — рабочая память удерживает ограниченное число элементов; мы уменьшаем лишнюю нагрузку через chunking и signaling.

2. **Testing Effect** — активное извлечение знаний из памяти улучшает запоминание сильнее, чем перечитывание.

3. **Generative Learning** — студент должен строить связи и выводы, а не пассивно читать.

## STEP 2: Generate Structured Content

Use the following structure. Each section should follow pedagogical best practices.

### Структура (СТРОГО СЛЕДУЙТЕ)

```markdown
### {title}

#### Ключевые понятия
- 3–5 ключевых терминов: по 1 короткому определению

- **Понятие 1**: Краткое определение (1 предложение)
- **Понятие 2**: Краткое определение (1 предложение)
- **Понятие 3**: Краткое определение (1 предложение)

---

#### Детальное объяснение
Короткие абзацы: 2–4 предложения.

##### Определение
Краткий абзац (2–4 предложения), объясняющий ключевой концепт.
Используйте **жирный шрифт** только для терминов при их ПЕРВОМ упоминании.

##### Разбор
Дополнительные детали и связи (2–4 предложения). Избегайте «стен текста».

##### Механизм/процесс (если применимо)
Как это работает? Какой механизм? (2–4 предложения)

#### Иллюстрации (Optional)
Если в исходном материале есть важные диаграммы, схемы или графики для этого раздела:
1. Найдите их в исходном PDF.
2. Вставьте маркер: `[[MEDIA_REF: {{"page": 5, "box_2d": [ymin, xmin, ymax, xmax], "caption": "Описание"}}]]`
   - `page`: номер страницы (начиная с 1)
   - `box_2d`: координаты [ymin, xmin, ymax, xmax] в масштабе 0-1000 (0=верх/лево, 1000=низ/право)
3. НЕ используйте markdown images `![alt](url)` напрямую.

---

#### Примеры

**Пример 1: [Название]**
Конкретный пример из реальной жизни или исходных материалов.
Объясните, почему этот пример иллюстрирует концепцию.

**Пример 2 (если уместно):**
Еще один пример, показывающий другой аспект.

---

#### Вопрос для самопроверки

**Вопрос:** [Открытый вопрос, требующий понимания]
- Примеры: "Как концепция X влияет на Y в реальных сценариях?"
- Примеры: "Почему подход A предпочтительнее подхода B в ситуации Z?"

**НЕ ДАВАЙТЕ ОТВЕТ ЗДЕСЬ.** Студент должен подумать сам.

#### Важные детали

> - Частых ошибок (что НЕ делать)

**Частые ошибки:**
- Ошибка 1: [Описание + почему это ошибка]
- Ошибка 2: [Описание + почему это ошибка]

**Связи с другими темами:**
[Если есть связь с предыдущими темами из {previous_context}, упомяните ее]

#### Резюме

**CRITICAL:** НЕ КОПИРУЙТЕ текст выше. Переформулируйте основные идеи в 3-4 новых предложениях.

Синтезируйте ключевые выводы. Покажите связи между концепциями. Ответьте на вопрос: "Почему это важно?"

[3-4 предложения, которые НЕ встречаются дословно в тексте выше]

---
```

### 🎨 Formatting Requirements (CRITICAL)

**✅ DO:**
- Use `###` для заголовка топика
- Use `####` для подразделов
- **Жирный шрифт** ТОЛЬКО для ключевых терминов (первое упоминание)
- Короткие параграфы: 2-4 предложения maximum
- Буллеты `-` для списков
- Нумерация `1.` для шагов/процедур
- Таблицы для сравнений (где уместно)
- Horizontal rules `---` между крупными секциями
- Code blocks ``` для кода, формул, structured data
- Blockquotes `> **Note:**` для важных замечаний

**❌ DON'T:**
- ❌ Стены текста >5 предложений без разбиения
- ❌ Выделение жирным **целых предложений** (визуальный шум)
- ❌ Использование плейсхолдеров `[Insert Definition Here]` — ВСЕГДА генерируйте реальный контент
- ❌ Копирование текста в резюме из разделов выше
- ❌ Вопросы с ответами сразу под ними (убивает testing effect)
- ❌ Эмодзи и декоративные символы в заголовках/структуре

## Content Guidelines

### Extracting Real Information
1. **Base content ONLY on materials provided** in {content_section}
2. **If materials lack specific details**, create reasonable educational content appropriate for {level}
3. **NEVER use placeholder text** — generate actual examples, definitions, explanations
4. **Maintain academic rigor** — cite mechanisms, causes, relationships

### Scannability (30-Second Test)
A student should grasp the main points by:
- Reading the 🎯 Ключевые понятия section (10 seconds)
- Scanning the headings and bold terms (10 seconds)
- Reading the 📝 Резюме (10 seconds)

### Length Calibration
- Target: {estimated_paragraphs} well-structured sections
- Quality > Quantity — better to have 3 excellent sections than 6 mediocre ones
- Each section = 2-4 sentences (roughly 50-100 words)
