## Scientific Framework

**Why this structure works:**

1. **Cognitive Load Theory (CLT)** — Rабочая память может удерживать только 3-5 единиц информации одновременно. Мы снижаем постороннюю нагрузку через chunking и signaling.

2. **Testing Effect** — Извлечение информации из памяти (active recall) повышает запоминание на 30-50% по сравнению с повторным чтением.

3. **Generative Learning** — Студент должен активно создавать связи, а не пассивно читать.

## STEP 1: Chain-of-Thought Planning (Internal)

**Before generating the formatted content, first plan internally:**

1. **Identify 3-5 Key Concepts** from the source materials
   - What are the core ideas this topic covers?
   - Which terms are fundamental vs. supporting?

2. **Design Self-Check Questions** (one per major concept)
   - Question must require UNDERSTANDING, not recall
   - Question should NOT be answerable by simply scanning the text
   - Example: "How does X relate to Y?" NOT "What is X?"

3. **Outline Logical Flow**
   - Progression: Definition → Explanation → Example → Application
   - Ensure each section builds on the previous one

4. **Identify Real-World Connections**
   - Where applicable, find practical examples from the materials

**Critical:** Keep this planning purely internal (mental). If you absolutely must output your thought process for clarity, enclose it strictly within `<thinking>...</thinking>` tags so it can be programmatically removed during parsing.

**After planning, proceed to STEP 2 to generate the formatted content.**

### 🎨 Formatting Requirements (CRITICAL)

**✅ DO:**
- Use `###` для заголовка топика
- Use `####` для подразделов
- **Жирный шрифт** ТОЛЬКО для ключевых терминов (первое упоминание)
- Короткие параграфы: 2-4 предложения maximum
- Буллеты `-` для списков
- Нумерация `1.` для шагов/процедур
- Таблицы для сравнений (где уместно)
- Horizontal rules `---` между крупными секциями
- Code blocks ``` для кода, формул, structured data
- Blockquotes `> **Note:**` для важных замечаний

**❌ DON'T:**
- ❌ Стены текста >5 предложений без разбиения
- ❌ Выделение жирным **целых предложений** (визуальный шум)
- ❌ Использование плейсхолдеров `[Insert Definition Here]` — ВСЕГДА генерируйте реальный контент
- ❌ Копирование текста в резюме из разделов выше
- ❌ Вопросы с ответами сразу под ними (убивает testing effect)
- ❌ Избыточное использование эмодзи (>2-3 на секцию)

## Content Guidelines

### Extracting Real Information
1. **Base content ONLY on materials provided** in {content_section}
2. **If materials lack specific details**, create reasonable educational content appropriate for {level}
3. **NEVER use placeholder text** — generate actual examples, definitions, explanations
4. **Maintain academic rigor** — cite mechanisms, causes, relationships

### Scannability (30-Second Test)
A student should grasp the main points by:
- Reading the 🎯 Ключевые понятия section (10 seconds)
- Scanning the headings and bold terms (10 seconds)
- Reading the 📝 Резюме (10 seconds)

### Length Calibration
- Target: {estimated_paragraphs} well-structured sections
- Quality > Quantity — better to have 3 excellent sections than 6 mediocre ones
- Each section = 2-4 sentences (roughly 50-100 words)
