---

## Scientific Framework

**Why outlining matters:**

1. **Advance Organizers (Ausubel)** — Seeing the structure BEFORE reading details activates prior knowledge and improves comprehension.

2. **Bloom's Taxonomy** — Outlining should progress from foundational concepts to advanced applications.

3. **Chunking (Miller)** — 3-8 main topics is the optimal range for working memory.

---

## Analysis Strategy

### 1. Identify Main Topics

**Main topics** are major sections or chapters, such as:
- "Introduction to X"
- "Core Principles of Y"
- "Applications of Z"

**Criteria:**
- 3-8 main topics (cognitive load management)
- Each topic represents a coherent concept or area
- Topics have clear boundaries (minimal overlap)

### 2. Extract Subtopics

**Subtopics** are specific concepts or sections within each main topic.

**Criteria:**
- 2-5 subtopics per main topic
- Each subtopic = one focused idea or skill
- Logical progression within each topic (simple → complex)

### 3. Ensure Logical Progression

**Overall flow should follow Bloom's Taxonomy:**
- **Remember/Understand** (early topics): Definitions, basic concepts
- **Apply** (middle topics): How to use/implement
- **Analyze/Evaluate** (later topics): Comparisons, advanced scenarios

**Example progression:**
1. Fundamentals of X
2. How X Works
3. Implementing X
4. Optimizing X
5. Advanced X Techniques

---

## Formatting Requirements

### Topic Naming

**✅ DO:**
- Keep topic names concise (max 6 words)
- Keep subtopic names concise (max 8 words)
- Use clear, descriptive language
- Use action-oriented language where appropriate ("Implementing...", "Understanding...")

**❌ DON'T:**
- Use vague names like "Miscellaneous" or "Other"
- Use overly long, complex titles
- Include chapter numbers if they're not semantic (e.g., "Chapter 3" → extract actual topic name)

### Hierarchical Structure

Use this format:

```
1. Main Topic Name
   - Subtopic 1
   - Subtopic 2
   - Subtopic 3
   
2. Second Main Topic
   - Subtopic 1
   - Subtopic 2
   ...
```

---

## Output Format

Return as a **plain text outline** (no JSON, no markdown formatting).

**Indentation:** Use exactly **3 spaces** before each subtopic dash to ensure consistent hierarchy.

Format:
```
1. [Main Topic 1]
   - [Subtopic 1.1]
   - [Subtopic 1.2]
   - [Subtopic 1.3]
   
2. [Main Topic 2]
   - [Subtopic 2.1]
   - [Subtopic 2.2]
   ...
   
3. [Main Topic 3]
   ...
```

**Requirements:**
- Numbered main topics (1, 2, 3...)
- Dash-prefixed subtopics (-)
- One blank line between main topics
- No extra formatting (no bold, no asterisks)

## Edge Cases

**If file content is sparse:**
- Create as few as 2-3 main topics
- Don't artificially inflate the outline

**If file is disorganized (e.g., lecture transcript):**
- Impose logical structure
- Group related concepts together
- Create foundational → advanced progression even if source doesn't follow it

**If file covers multiple unrelated subjects:**
- Focus on the PRIMARY subject ({subject})
- Group secondary topics under a final "Additional Topics" section if necessary
