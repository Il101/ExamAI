## Your Task: Professional Finalization

### 1. Create Title Page

Add a professional title page at the very beginning with:

```markdown
# Study Guide: [Real Subject Name]

**AI-Generated Summary**

**Subject:** [Infer from ACTUAL content, ignore generic title]
```

**Important:** Determine the REAL subject from the content. If the course is titled "Materials" but covers Python programming, write "Subject: Python Programming Fundamentals".

---

### 2. Table of Contents

Create a comprehensive TOC with all {section_count} topics:

```markdown
## 📚 Table of Contents

1. [Topic 1 Title](#topic-1-title)
2. [Topic 2 Title](#topic-2-title)
...
{section_count}. [Final Topic](#final-topic)

---
```

**Technical:** Use GitHub-style anchor links:
- Lowercase all letters
- Replace spaces with hyphens
- Remove punctuation
- Example: `### My Header!` → `[My Header!](#my-header)`

### 3. Polish Formatting

**CRITICAL INSTRUCTION:** Keep the **main body content** of the topics mostly AS IS, unless there are formatting errors or placeholders. 

Your job is to **polish the structure**, not rewrite the content. Focus your creative energy on:
- Title Page (new)
- Table of Contents (new)
- Comprehensive Summary (new, generative)
- Key Terms Glossary (new)
- Self-Check Questions (new)

**Review and fix formatting issues only:**

#### Consistency
- [ ] All topic headings use `###` (level 3)
- [ ] All subsections use `####` (level 4)
- [ ] Proper spacing between sections (one blank line before headings, two blank lines between major sections)
- [ ] Horizontal rules `---` separate major topics

#### Visual Clarity
- [ ] Bold used ONLY for key terms (not entire sentences)
- [ ] Lists are properly formatted (`-` for bullets, `1.` for numbers)
- [ ] Tables render correctly (proper alignment with `|---|---|`)
- [ ] Code blocks use proper syntax highlighting (```python, ```bash, etc.)
- [ ] Blockquotes `>` are used consistently for important notes

#### Content Quality
- [ ] **Remove ALL placeholder text** (`[Insert X]`, `[TBD]`, etc.)
- [ ] Remove duplicate information (if same concept explained twice)
- [ ] Ensure terminology is consistent (don't say "function" in one section and "method" in another for same concept)
- [ ] Add smooth transitions between topics where logical (e.g., "Building on the concept of X from the previous section...")

### 4. Generate Final Summary

**At the very end, add a comprehensive summary section:**

```markdown
---

## 📝 Comprehensive Summary

> **Generative Summarizing** — This section synthesizes key insights across all topics.

[CRITICAL INSTRUCTION HERE — see below]

---
```

**CRITICAL - Generative Summarizing Rules:**

1. **DO NOT copy text** from any section above
2. **REPHRASE** the main ideas in new sentences
3. **SYNTHESIZE** connections between topics:
   - "Topic A's concept of X directly enables Topic C's Y..."
   - "The progression from [early topic] to [later topic] shows..."
4. **Write 4-6 paragraphs** (roughly 300-400 words total)
5. **Answer the question:** "What are the big-picture takeaways from this entire course of study?"

**Scientific basis:** Generative summarizing forces deeper cognitive processing than extractive summarizing (Mayer, 2016). By rephrasing, you internalize rather than just recognize.

---

### 5. Create Key Terms Glossary (Cornell Method)

**After the summary, add:**

```markdown
## 📖 Key Terms Glossary

> **Cornell Method** — Quick reference for review

| Term | Definition |
|------|-----------|
| **Term 1** | Concise definition in your own words (not copy-pasted from content) |
| **Term 2** | Another definition... |
| ... | ... |

*Note: These definitions are rephrased for better retention.*

---
```

**Instructions:**
- Identify 15-25 most important terms across ALL topics
- Define each in **your own words** (rephrase from content, don't copy)
- Keep definitions to 1-2 sentences (under 30 words)
- Alphabetical order for easy lookup

### 6. Add Self-Check Questions

**Finally, add a comprehensive self-assessment section:**

```markdown
## ❓ Self-Check Questions

> Test your understanding across all topics

Answer these questions WITHOUT looking back at the notes first. They're designed to test comprehension, not memorization.

### Recall Questions (Basic)
1. [Question about fundamental concept from Topic 1]
2. [Question about key term from Topic 3]
...

### Application Questions (Medium)
6. [Scenario-based question requiring understanding]
7. [Question asking to apply concept to new situation]
...

### Analysis Questions (Advanced)
11. [Question requiring comparison/evaluation]
12. [Question asking "why" or "how does X relate to Y"]

**Answers:**
[Provide brief answers at the end, after a page break or under a collapsible section]

---
```

**Guidelines:**
- Create 10-15 questions total
- Mix difficulty: 40% recall, 40% application, 20% analysis
- Cover different topics (not all from one section)
- Questions should require understanding, not just memorization

## Formatting Guidelines

### ✅ DO:
- Use consistent heading levels (`###` for topics, `####` for subsections)
- Add horizontal rules `---` between major sections
- Ensure all lists are properly formatted
- Use tables for comparisons
- Use emojis sparingly for visual navigation (📚 📝 ❓ 💡 but max 1 per heading)
- Keep paragraphs SHORT (2-4 sentences)

### ❌ DON'T:
- Change factual content (you're an editor, not a content creator)
- Add new topics or remove existing ones
- Use placeholder text ANYWHERE
- Copy-paste sentences into the summary (must rephrase!)
- Create a "wall of text" without structure
