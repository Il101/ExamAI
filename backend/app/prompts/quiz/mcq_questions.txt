## Scientific Framework

**Why MCQs with explanations work for learning:**

1. **Testing Effect** — The act of retrieving an answer (even if wrong) strengthens memory more than re-reading.

2. **Elaborative Interrogation** — Understanding *why* an answer is correct (or why distractors are wrong) creates deeper neural encoding than simple yes/no feedback.

3. **Metacognition** — When students realize their misconceptions (by understanding why they chose the wrong answer), they update their mental models.

**Research:** Roediger & Marsh (2005) showed that testing with feedback produces 30-50% better retention than passive review.

## Question Design Principles

### 1. Question Stem (The Question Itself)

**✅ DO:**
- Focus on **understanding and application**, not rote memorization
- Use **scenario-based questions** where possible:
  - "A developer needs to store user passwords. Which approach is most secure?"
  - NOT: "What is bcrypt?"
- Be specific and clear (avoid "all of the following EXCEPT...")
- Keep stem under 40 words

**❌ DON'T:**
- "Which of the following is true?" (too vague)
- Double negatives: "Which is NOT incorrect?" (confusing)
- Trick questions that test reading comprehension, not knowledge
- **"All of the above" / "None of the above"** — NEVER use these (they measure test-taking skills, not knowledge, and LLMs often generate logically inconsistent options with these)

**Difficulty levels:**
- **Recall (30%):** "What is the purpose of X?"
- **Application (50%):** "In scenario Y, which approach would work best?"
- **Analysis (20%):** "Why is approach A superior to approach B for use case Z?"

---

### 2. Answer Options

**Format:** 4 options (A, B, C, D)
- **1 correct answer**
- **3 plausible distractors** (wrong answers)

**✅ Effective distractors:**
- Based on **common misconceptions** or partial understanding
- Similar length and structure to the correct answer
- Grammatically consistent with the stem
- Not obviously wrong

**❌ Poor distractors:**
- Randomly generated nonsense
- Joke answers or obvious absurdities
- "All of the above" / "None of the above" (never use)

**Example:**
```
Question: A Python function needs to process data but shouldn't modify the original list. What should the developer do?

A) Pass the list directly and use .append() carefully
B) Create a shallow copy with list.copy() before processing
C) Use a global variable to track changes
D) Pass the list as *args to prevent modification

Correct: B
Why B is correct: list.copy() creates a new list object, so modifications don't affect the original.
Why A is wrong: Even careful use of .append() modifies the original (misconception: "being careful" prevents side effects)
Why C is wrong: Global variables don't prevent modification, they make debugging harder
Why D is wrong: *args just unpacks elements, doesn't create a copy
```

---

### 3. Explanations (CRITICAL COMPONENT)

**This is what turns testing into learning.**

For EACH question, provide:

1. **Why the correct answer is correct** (2-3 sentences)
   - Explain the mechanism/principle
   - Connect to broader concept if applicable

2. **Why key distractors are wrong** (1 sentence each, focus on 1-2 most tempting ones)
   - Address the misconception that makes this answer tempting
   - Use phrases like "This is a common mistake because..."

**Example explanation:**
```
Why B is correct:
list.copy() creates a shallow copy, which is a new list object with the same elements. Any modifications (append, remove, etc.) affect only the copy, leaving the original unchanged. This is essential for functions that need to work with data without side effects.

Why A is wrong:
Even with careful coding, .append() directly modifies the list object in memory. There's no way to "carefully" avoid this — it's how Python lists work (they're mutable).

Why D is tempting but wrong:
*args syntax is for variable-length arguments in function definitions, not for copying data structures.
```

---

### 4. Difficulty Distribution

Aim for Bloom's Taxonomy progression:

| Level | % of Questions | Example |
|-------|---------------|---------|
| **Remember** (Recall) | 30% | "What does X mean?" |
| **Understand** (Application) | 50% | "How would you use X in scenario Y?" |
| **Analyze/Evaluate** | 20% | "Why is approach A better than B for Z?" |

## Output Format

Return as **JSON array** (no markdown code blocks, raw JSON only):

```json
[
  {
    "question": "Question stem here",
    "options": {
      "A": "First option",
      "B": "Second option",
      "C": "Third option",
      "D": "Fourth option"
    },
    "correct": "B",
    "explanation": {
      "correct": "Why B is the right answer (2-3 sentences)",
      "distractors": {
        "A": "Why A is wrong (common misconception)",
        "C": "Why C is wrong (if it's a tempting distractor)"
      }
    },
    "difficulty": "recall|application|analysis",
    "tags": ["tag1", "tag2"]
  },
  ...
]
```

**Fields:**
- `question`: The question stem (max 40 words)
- `options`: Object with A/B/C/D keys (each option <15 words)
- `correct`: Letter of correct answer ("A", "B", "C", or "D")
- `explanation.correct`: Why correct answer is right (2-3 sentences, ~50 words)
- `explanation.distractors`: Why wrong answers are wrong (focus on 1-2 key ones)
- `difficulty`: One of `"recall"`, `"application"`, `"analysis"`
- `tags`: 1-3 topic tags

## Quality Checklist

Before outputting, verify EACH question:

**Pedagogy:**
- [ ] Question tests understanding, not trivial facts
- [ ] Explanation for correct answer teaches the principle
- [ ] At least 1 distractor explanation addresses a misconception
- [ ] Difficulty level is appropriate and marked

**Format:**
- [ ] Question stem is clear and under 40 words
- [ ] All 4 options are similar length (no obvious outliers)
- [ ] No grammatical clues to correct answer
- [ ] JSON is valid

**Distribution:**
- [ ] ~30% recall, ~50% application, ~20% analysis
- [ ] Questions cover different aspects of content

---

## Edge Cases

**If content is insufficient for {num_questions} questions:**
- Create as many as the content supports
- Do NOT invent facts outside source material
- Prefer fewer high-quality questions over many shallow ones

**If a concept has only one obvious question:**
- Rephrase from different angles (definition vs. application vs. comparison)
- Use scenario variations
